# AgentFlow Dashboard

![AgentFlow Dashboard](https://img.shields.io/badge/Status-Active-success.svg)
![License](https://img.shields.io/badge/License-MIT-blue.svg)

AgentFlow Dashboard is a modern, Jira-like task management and team collaboration platform built for speed and security. It provides a comprehensive suite of tools for managing projects, tracking task statuses, assigning multiple users, discussing issues, and maintaining detailed audit trails.

## 🚀 Key Features

* **Advanced Task Management:** Create, update, and manage tasks with standard statuses (TODO, IN PROGRESS, DONE, etc.) and priorities.
* **Collaboration & Mentions:** Integrated comment system with `@username` mentions and real-time notifications.
* **Audit Logging & Security:** Comprehensive, tamper-proof audit logs tracking all critical actions (logins, task modifications, IP addresses).
* **IP Blocking & Rate Limiting:** Built-in protection against brute-force attacks and malicious actors with admin-level IP banning.
* **Role-Based Access Control (RBAC):** Distinct `ADMIN` and `MEMBER` roles with a manual approval process for new users.
* **Optimistic UI:** Lightning-fast Next.js frontend with optimistic updates for a premium user experience.

## 🛠 Tech Stack

* **Framework:** Next.js (App Router, Server Actions)
* **Language:** TypeScript
* **Database ORM:** Prisma
* **Database:** SQLite (Easily migratable to PostgreSQL for enterprise scaling)
* **Styling:** CSS Modules / Vanilla CSS
* **Icons:** Lucide React

## 📦 Getting Started

### Prerequisites
* Node.js (v18+)
* npm or yarn

### Installation

1. Clone the repository and install dependencies:
   ```bash
   npm install
   ```

2. Initialize the database schema:
   ```bash
   npx prisma db push
   ```

3. Run the development server:
   ```bash
   npm run dev
   ```
   The application will be available at `http://localhost:5000`.

4. Run the production server:
   ```bash
   npm run build
   npm start
   ```
   The production server listens on port `80`, so your domain can point directly to the VPS public IP. On Linux, make sure no other service such as Nginx or Apache is already using port `80`.

## 🔐 Authentication & Onboarding

1. **Initial Admin Setup:** When the application is launched, a default admin account is automatically created:
   * **Email:** `admin@admin.com`
   * **Password:** `admin123`
   *(You will be forced to change this password upon your first login).*

2. **User Registration:** To register new users, you need the **Registration Token** generated by the server. Check your server logs (e.g., `pm2 logs` or your console output) for a line that looks like:
   `[AgentFlowDashboard] Registration Token: ...`
   All newly registered users (after the first admin) require admin approval before they can log in.

## 🛡 Security Architecture

This application incorporates enterprise-grade security headers (X-Frame-Options, X-Content-Type-Options) and in-memory rate limiting. All administrative actions and suspicious activities (like failed logins) are securely recorded in the `AuditLog` table, which is accessible via the restricted `Audyty` (Audit) tab on the frontend.
